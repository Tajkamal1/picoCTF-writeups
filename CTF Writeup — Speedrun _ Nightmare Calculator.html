Challenge Description

A TCP service on 142.93.12.237:1337 (DigitalOcean, North Bergen NJ) presents a "nightmare calculator". On every connection it sends 256 math equations and demands answers within 20 seconds total. The twist: operators are remapped randomly at connection time and again every ~9 rounds mid-game via a PROTOCOL UPDATE message.

02

Reconnaissance

Three distinct problems were identified after initial testing:

Dynamic operator remapping — symbols like '$', '#', '*' change meaning each session. Can't hardcode anything.
Mid-game PROTOCOL UPDATE — after ~9 rounds, the server sends a fresh RULES: line with entirely new mappings. Old symbols become invalid.
RTT wall — ping 142.93.12.237 from Hyderabad showed ~267ms. At one request per round: 256 × 267ms = 68 seconds minimum. No code optimization can fix a physics problem.
03

Solution Design

The solution has two independent parts: a correct parser/evaluator, and running it close enough to the server that the RTT budget fits in 20 seconds.

Parser: Parse the RULES: line with a regex into a dict[symbol → python_op]. For each equation, walk character-by-character and swap custom symbols for their Python equivalents, then call eval(). Re-parse whenever a new RULES: line is seen (clearing the old map first).

Speed: Use a buffered socket that greedily drains all queued server data after each send. This turns N individual RTTs into batch RTTs. For the remaining RTT problem, run from a VPS co-located with the challenge server (GitHub Codespaces East US, ~10ms).

04

Bugs Encountered

Mid-game rule updates ignored — initial code used a RULES_PARSED=1 flag that blocked any re-parsing. Fixed by always re-parsing on every RULES: line and clearing the map with unset OP_MAP; declare -A OP_MAP.
Banner/question in separate TCP segments — the batch drain used an 80ms timeout that expired before the server sent the first question, causing an immediate abort. Fixed with a two-phase read: line-by-line until round 1 is answered, then switch to greedy batch mode.
Bash subprocess overhead — the bash solver spawned a new python3 -c process per round (~50ms startup × 512 calls = 25s minimum). Replaced entirely with a single Python process.
05

Final Exploit Code

Run from GitHub Codespaces (East US) or any VPS near New Jersey. From local Hyderabad machine it will time out at ~55 rounds. 
PYTHON#!/usr/bin/env python3
# Nightmare Calculator — Final Solver
# NovrozCTF 2025  |  Misc / Speedrun  |  500pts
# Run from a VPS close to North Bergen, NJ (DigitalOcean NYC)

import socket, re, time

HOST = "142.93.12.237"
PORT = 1337

OP_TO_PY = {"ADD":"+", "SUB":"-", "MUL":"*", "MOD":"%", "XOR":"^", "DIV":"//"}

def parse_rules(line):
    op_map = {}
    for m in re.finditer(r"'(.)'[^A-Z]*([A-Z]+)", line):
        sym, name = m.group(1), m.group(2)
        if name in OP_TO_PY:
            op_map[sym] = OP_TO_PY[name]
    return op_map

def translate(expr, op_map):
    return "".join(op_map.get(ch, ch) for ch in expr)


class BufSocket:
    def __init__(self, host, port):
        self.s = socket.socket()
        self.s.connect((host, port))
        self.buf = b""

    def _fill(self, timeout):
        self.s.settimeout(timeout)
        try:
            chunk = self.s.recv(65536)
            if not chunk: return False
            self.buf += chunk; return True
        except socket.timeout: return False

    def readline(self, timeout=8.0):
        deadline = time.perf_counter() + timeout
        while b"\n" not in self.buf:
            remaining = deadline - time.perf_counter()
            if remaining <= 0: return None
            if not self._fill(min(remaining, 1.0)):
                if b"\n" not in self.buf: return None
        line, self.buf = self.buf.split(b"\n", 1)
        return line.decode(errors="replace").rstrip("\r")

    def drain_lines(self, batch_timeout=0.08):
        lines = []
        while b"\n" in self.buf:
            line, self.buf = self.buf.split(b"\n", 1)
            lines.append(line.decode(errors="replace").rstrip("\r"))
        if self._fill(batch_timeout):
            while b"\n" in self.buf:
                line, self.buf = self.buf.split(b"\n", 1)
                lines.append(line.decode(errors="replace").rstrip("\r"))
        return lines

    def sendbatch(self, answers):
        if answers:
            self.s.sendall(("\n".join(answers) + "\n").encode())

    def drain_all(self, timeout=4.0):
        lines = []
        while self._fill(timeout):
            while b"\n" in self.buf:
                line, self.buf = self.buf.split(b"\n", 1)
                lines.append(line.decode(errors="replace").rstrip("\r"))
            timeout = 1.0
        return lines

    def close(self): self.s.close()


def process_line(line, op_map, round_num):
    answer = None
    if "RULES:" in line:
        new_map = parse_rules(line)
        if new_map:
            op_map = new_map
            print(f"[~] Rules reloaded: {op_map}")
    elif "Calculate:" in line:
        round_num += 1
        expr = line.split("Calculate:", 1)[1].strip()
        try:
            answer = str(eval(translate(expr, op_map)))
        except Exception as e:
            print(f"[ERR {round_num}] {e}"); answer = "0"
        print(f"[{round_num:>3}/256] {expr[:60]:<60} = {answer}")
    elif "novruzctf{" in line:
        print(f"\n{'='*55}\n  FLAG: {line.strip()}\n{'='*55}\n")
    return op_map, round_num, answer


def main():
    print(f"[*] Connecting to {HOST}:{PORT}")
    conn = BufSocket(HOST, PORT)
    print("[+] Connected\n")
    op_map, round_num = {}, 0
    t0 = time.perf_counter()

    try:
        # Phase 1: read line-by-line until round 1 answered
        while round_num == 0:
            line = conn.readline(timeout=8.0)
            if line is None: return
            op_map, round_num, answer = process_line(line, op_map, round_num)
            if answer: conn.sendbatch([answer])

        # Phase 2: greedy batch drain → bulk answer
        while round_num < 256:
            lines = conn.drain_lines(batch_timeout=0.08)
            if not lines:
                line = conn.readline(timeout=6.0)
                if line is None: break
                lines = [line]
            answers = []
            for line in lines:
                if not line.strip(): continue
                op_map, round_num, answer = process_line(line, op_map, round_num)
                if answer: answers.append(answer)
                if round_num >= 256: break
            conn.sendbatch(answers)

    finally:
        for line in conn.drain_all(timeout=4.0):
            if line.strip():
                process_line(line, op_map, round_num)
                if line.strip(): print(f"[srv] {line}")
        elapsed = time.perf_counter() - t0
        print(f"\nRounds: {round_num}/256  |  Time: {elapsed:.2f}s")
        conn.close()

if __name__ == "__main__":
    main()

Key Takeaways

Parse, don't hardcode. Every operator is a variable. The RULES: regex handles all 5 operator names across all sessions.
Mid-game state changes are a classic CTF trap. Always check whether a server can invalidate your assumptions mid-session.
RTT is a first-class constraint. When a challenge has a strict time limit and requires many round-trips, check the ping before optimizing code. Proximity to the server was the actual solution.
Avoid subprocess overhead in tight loops. Spawning python3 -c per round costs ~50ms each — more than the allowed budget per round at low RTT.
